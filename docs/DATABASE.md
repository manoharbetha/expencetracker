# MongoDB Database Schemas

Fintell communicates with a backend powered by MongoDB Atlas. Although MongoDB is schemaless, the FastAPI backend applies validation and constraints using PyDantic schemas. Below are the core collections and document structures.

---

## 1. `users`
Tracks user credentials, profile configuration, and settings.
```json
{
  "_id": "ObjectId('...')",
  "name": "Aarav Mehta",
  "email": "aarav@example.com",
  "hashedPassword": "$2b$12$...",
  "monthlyIncome": 75000,
  "currency": "INR",
  "country": "IN",
  "createdAt": "2026-06-25T14:32:00.000Z"
}
```

---

## 2. `expenses`
Stores individual transactions. Linked to `users` and optionally `credit_cards`.
```json
{
  "_id": "ObjectId('...')",
  "userId": "ObjectId('...')",
  "amount": 1500,
  "category": "Food",
  "description": "Lunch with team",
  "paymentMethod": "Credit Card",
  "creditCardId": "ObjectId('...')",
  "date": "2026-06-28",
  "createdAt": "2026-06-28T09:12:00.000Z",
  "updatedAt": "2026-06-28T09:12:00.000Z"
}
```

---

## 3. `credit_cards`
Tracks configured credit cards and outstanding debts.
```json
{
  "_id": "ObjectId('...')",
  "userId": "ObjectId('...')",
  "cardName": "HDFC Millennia",
  "bankName": "HDFC Bank",
  "creditLimit": 150000,
  "createdAt": "2026-06-26T10:15:00.000Z"
}
```
*Note: Available limit, outstanding balance, and usage percentage are computed dynamically on the backend or service layer by aggregating the card's linked transactions in `expenses`.*

---

## 4. `goals`
Tracks savings goals milestones.
```json
{
  "_id": "ObjectId('...')",
  "userId": "ObjectId('...')",
  "goalName": "MacBook Pro Fund",
  "targetAmount": 120000,
  "savedAmount": 30000,
  "deadline": "2026-12-31",
  "createdAt": "2026-06-27T08:00:00.000Z"
}
```

---

## 5. `debts`
Stores outstanding general loans or lent/borrowed liabilities.
```json
{
  "_id": "ObjectId('...')",
  "userId": "ObjectId('...')",
  "title": "Home Loan Principal",
  "amount": 500000,
  "interestRate": 8.5,
  "emi": 15000,
  "dueDate": "2026-07-10",
  "type": "borrowed"
}
```

---

## 6. `notifications`
Stores in-app alerts generated by backend validation rules.
```json
{
  "_id": "ObjectId('...')",
  "userId": "ObjectId('...')",
  "title": "High Credit Card Utilization",
  "message": "Your HDFC Millennia card utilization has exceeded 80%.",
  "type": "credit_card",
  "category": "credit_card",
  "priority": "high",
  "isRead": false,
  "createdAt": "2026-06-28T11:00:00.000Z"
}
```
